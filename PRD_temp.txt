# PRD: NGO Mentorship Management Platform

**Version:** 1.0  
**Date:** 2026-03-30  
**Tech Stack:** InsForge.dev (Backend + Database + Auth), React/Next.js (Frontend)

---

## 1. Executive Summary

A web-based platform connecting NGOs, mentors, and students (ages 1–19) for structured mentorship programs. The system enables automatic mentor-student matching based on skills and needs, tracks progress through online/offline hybrid sessions, and provides comprehensive analytics for NGO administrators.

---

## 2. User Personas & Roles

| Role | Permissions | Key Actions |
|------|-------------|-------------|
| **NGO Admin** | Full system access | Upload datasets, manage mentors, view analytics, assign students, monitor progress, remove underperforming mentors |
| **Mentor** | Assigned students only | View mentees, conduct sessions, upload progress, access training materials, chat with students |
| **Student** | View assigned mentor, access resources | View mentor details, access learning portal, submit assignments, chat with mentor |

---

## 3. Core Workflows

### 3.1 NGO Admin Workflow

```
Login → Dashboard (Analytics Overview)
├── Add Mentors (Manual entry with skills/details)
├── Training Center (Upload videos/PDFs → Mentors complete → Flag as "Trained")
├── Auto-Assignment Engine
│   ├── Upload Student Excel
│   ├── Upload Mentor Excel
│   ├── System matches: Skills ↔ Needs + Gender Pref + Timing + Location
│   ├── Assigned Students → Mentor Dashboard
│   └── Unmatched → "Others Queue" (Manual assignment pool)
├── Monitor Progress (View all mentor-student pairs, session logs, performance)
└── Reports (Download PDF/Excel analytics)
```

### 3.2 Mentor Workflow

```
Login → Dashboard
├── Training Module (Watch videos/PDFs → Mark complete → Become "Active")
├── My Mentees (View assigned students, duration, progress history)
├── Session Management
│   ├── Weekdays: Online sessions (Video calls + Chat)
│   ├── Weekends: Offline visits (Download resources → Visit school → Conduct class → Upload quiz results)
│   └── Progress Update (Upload scores, attendance, notes)
├── Learning Resources (Access training materials to share with students)
└── Chat (1-on-1 with students + Group chat with all mentees)
```

### 3.3 Student Workflow

```
Login → Dashboard
├── My Mentor (View details, contact, schedule)
├── Learning Portal (Access videos/PDFs shared by mentor - if mobile/internet available)
├── Assignments (View tasks, submit work)
├── Progress (View own scores, attendance)
└── Chat (Message mentor, participate in group discussions)
```

## 6. Auto-Assignment Algorithm

### 6.1 Matching Logic (Priority Order)

```javascript
function calculateMatchScore(student, mentor) {
  let score = 0;
  let factors = {};

  // Priority 1: Skills–Needs Match (50% weight)
  const skillOverlap = student.needs.filter(need => mentor.skills.includes(need)).length;
  if (skillOverlap > 0) {
    score += 50 * (skillOverlap / student.needs.length);
    factors.skillMatch = true;
  } else {
    factors.skillMatch = false;
  }

  // Priority 2: Gender Preference (20% weight)
  if (!student.preferredMentorGender ||
      student.preferredMentorGender === 'no_preference' ||
      student.preferredMentorGender === mentor.gender) {
    score += 20;
    factors.genderPreferenceMet = true;
  } else {
    factors.genderPreferenceMet = false;
  }

  // Priority 3: Timing Alignment (15% weight)
  const studentTiming = student.mobileAccessible ? 'weekday_online' : 'weekend_offline';
  if (mentor.preferredTiming === 'both' || mentor.preferredTiming === studentTiming) {
    score += 15;
    factors.timingAligned = true;
  } else {
    factors.timingAligned = false;
  }

  // Priority 4: Location Compatibility (15% weight)
  if (student.location.city === mentor.location.city) {
    score += 15;
    factors.locationCompatible = true;
  } else if (student.location.area === mentor.location.area) {
    score += 10; // Same urban/rural type
    factors.locationCompatible = 'partial';
  } else {
    factors.locationCompatible = false;
  }

  return { score, factors };
}

function autoAssignStudents(students, mentors) {
  const assignments = [];
  const othersQueue = [];

  const availableMentors = mentors.filter(m =>
    m.isTrained && m.status === 'active' && m.currentStudents < m.maxStudents
  );

  for (const student of students) {
    if (student.assignedMentorId) continue;

    let bestMatch = null;
    let bestScore = 0;

    for (const mentor of availableMentors) {
      const { score, factors } = calculateMatchScore(student, mentor);
      if (score > bestScore && score >= 60) { // Minimum 60% match threshold
        bestScore = score;
        bestMatch = { mentor, score, factors };
      }
    }

    if (bestMatch) {
      assignments.push({
        studentId: student.id,
        mentorId: bestMatch.mentor.id,
        score: bestMatch.score,
        factors: bestMatch.factors,
        type: 'auto'
      });
      bestMatch.mentor.currentStudents++;
    } else {
      othersQueue.push({
        studentId: student.id,
        reason: determineReason(student, availableMentors),
        tempActivityType: assignTempActivity(student.needs)
      });
    }
  }

  return { assignments, othersQueue };
}

function assignTempActivity(needs) {
  if (needs.includes('Mental Health')) return 'mental_health';
  if (needs.includes('Cultural')) return 'cultural';
  return 'ethics_values';
}
```

## 9. Mentor Lifecycle & Performance Management

### 9.1 Status Flow

```
Pending Training → Active → [Warning] → [Suspended] → Removed
                      ↓          ↓            ↓
               Good Performance  Performance  Appeal
               Review (3 months) Review (1 month) Window
```

### 9.2 Performance Metrics (Auto-calculated)

- **Session Completion Rate:** % of scheduled sessions completed
- **Student Progress:** Average improvement in student metrics
- **Attendance Punctuality:** On-time session starts
- **Response Time:** Speed of responding to student messages
- **Student Feedback:** Ratings from students (age-appropriate)

### 9.3 Removal Triggers

| Trigger | Action | Timeline |
|---|---|---|
| Session completion < 60% | Warning | Immediate |
| No activity for 14 days | Warning | After 14 days |
| 2+ student complaints | Suspension | Immediate |
| Performance rating < 2.5 for 4 weeks | Suspension | After review |
| Failed to improve after warning | Removal | 30 days post-warning |

### 9.4 Appeal Process

- Suspended mentors can submit an appeal with justification
- NGO admin reviews appeal alongside performance data
- Decision: Reinstate (probation) or confirm removal

## 11. Security & Compliance

### 11.1 Data Protection

- **Encryption:** All data encrypted at rest and in transit (InsForge default)
- **Access Control:** Strict Role-Based Access Control (RBAC) enforcement
- **PII Handling:** Minimal collection; no sensitive personal data stored for minors
- **Audit Logs:** All admin actions logged (mentor removals, assignments, data exports)

### 11.2 Content Safety

- Training materials reviewed by NGO before upload
- Chat monitoring with flagging system for inappropriate content
- Report abuse button available to students and mentors
